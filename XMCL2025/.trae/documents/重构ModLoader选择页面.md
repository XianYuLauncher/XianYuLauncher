# 重构ModLoader选择页面计划

## 1. 问题分析
当前ModLoader选择页面使用两个独立的ListView控件，存在以下问题：
- UI设计不符合规范，不美观
- 加载器与版本列表优先级不明确
- 用户体验不佳，难以理解选择关系
- 加载状态显示位置不合理
- 可能存在重复加载问题

## 2. 解决方案
改用Expander控件重构页面，每个ModLoader作为一个可展开的Expander，版本列表显示在Expander内部。

## 3. 实现细节

### 3.1 XAML布局重构
- **移除**当前的两个ListView控件
- **添加**Expander控件集合，每个ModLoader对应一个Expander
- **设计**Expander头部，包含图标区域和ModLoader名称
- **在Expander内部**添加版本列表
- **整合**加载指示器到Expander内部
- **调整**垂直长度，确保布局合理

### 3.2 数据结构调整
- **ViewModel中添加**ModLoader项的模型类，包含加载状态、版本列表等属性
- **修改**AvailableModLoaders为ModLoader项的ObservableCollection
- **添加**每个ModLoader的加载状态管理

### 3.3 交互逻辑实现
- **点击Expander头部**选择对应的ModLoader
- **点击版本项**选择具体版本
- **实现**加载状态管理，仅在首次展开或页面重新进入时加载
- **添加**详细的加载指示器，显示在Expander内部

### 3.4 性能优化
- **避免**重复加载，仅在需要时加载版本列表
- **实现**加载状态缓存，提升用户体验
- **优化**数据绑定，减少不必要的UI更新

## 4. 预期效果
- 更美观、符合UI规范的界面设计
- 清晰的ModLoader与版本层级关系
- 直观的交互体验
- 合理的加载状态显示
- 优化的性能表现

## 5. 实现步骤
1. 修改XAML布局，添加Expander控件
2. 调整ViewModel数据结构
3. 实现Expander展开/折叠逻辑
4. 实现版本列表加载与显示
5. 添加加载状态管理
6. 优化交互体验
7. 测试功能完整性

## 6. 技术要点
- 使用Expander控件实现可折叠布局
- 实现延迟加载机制
- 添加加载状态指示
- 优化数据绑定性能
- 实现直观的选择交互
