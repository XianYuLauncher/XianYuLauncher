# 版本列表页面实现计划

## 1. 设计概述

### 1.1 页面定位
- 新增"版本列表"页面，从启动页面通过导航入口访问
- 清晰展示所有已安装的Minecraft版本信息
- 为后续版本管理功能（打开文件夹、导出整合包等）奠定基础

### 1.2 核心功能
- 显示已安装版本列表
- 版本信息包括：名称、类型、安装日期等
- 与现有导航系统无缝集成
- 保持与启动器整体设计风格一致

## 2. 实现步骤

### 2.1 创建页面文件
1. **创建ViewModels/版本列表ViewModel.cs**：实现版本列表的数据逻辑
2. **创建Views/版本列表Page.xaml**：实现版本列表的UI布局
3. **创建Views/版本列表Page.xaml.cs**：实现页面的代码-behind逻辑

### 2.2 配置导航系统
1. **修改Services/PageService.cs**：注册新页面
2. **修改Views/ShellPage.xaml**：添加导航菜单项
3. **更新资源文件**：添加页面标题和描述

### 2.3 实现ViewModel逻辑
1. 继承ObservableRecipient类
2. 注入IMinecraftVersionService和IFileService
3. 实现版本信息获取逻辑
4. 定义版本信息模型（包含名称、类型、安装日期等）
5. 实现版本列表的刷新功能

### 2.4 设计页面布局
1. 使用Grid布局组织页面结构
2. 顶部显示页面标题
3. 主体使用ListView或GridView展示版本列表
4. 每个版本项显示：
   - 版本名称
   - 版本类型（Release/Snapshot/Beta/Alpha）
   - 安装日期
   - 版本号
5. 添加刷新按钮

### 2.5 数据绑定
1. 将版本列表绑定到ListView/GridView
2. 实现版本项的模板
3. 绑定刷新命令

## 3. 技术细节

### 3.1 版本信息获取
- 复用现有IMinecraftVersionService的GetInstalledVersionsAsync方法
- 扩展版本信息，添加安装日期等元数据
- 通过文件系统获取版本文件夹的创建日期

### 3.2 页面布局设计
- 遵循WinUI设计规范
- 使用与现有页面一致的样式和主题
- 确保响应式设计，适配不同屏幕尺寸

### 3.3 导航集成
- 在ShellPage.xaml的NavigationView.MenuItems中添加新的NavigationViewItem
- 设置正确的helpers:NavigationHelper.NavigateTo属性
- 添加适当的图标和标题

## 4. 预期效果

### 4.1 页面外观
- 清晰的版本列表展示
- 现代化的WinUI设计风格
- 与现有页面一致的视觉体验

### 4.2 功能实现
- 正确显示所有已安装版本
- 支持版本列表刷新
- 与现有导航系统无缝集成
- 为后续版本管理功能预留扩展点

## 5. 后续扩展考虑

- 版本项右键菜单（打开文件夹、删除版本等）

- 版本信息获取可能因文件系统权限问题失败
- 大量版本时的性能问题
- 与现有功能的兼容性问题

- 验证导航功能正常
- 验证刷新功能正常
- 验证在不同屏幕尺寸下的显示效果
- 验证与现有功能无冲突
