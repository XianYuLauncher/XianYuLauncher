# 新手教程页面实现方案

## 1. 页面结构设计

### 1.1 主页面布局
- **SelectorBar**：顶部导航控件，用于切换三个教程页面
- **内容区域**：动态显示当前选中的教程内容
- **导航按钮**：底部包含"上一步"和"下一步"按钮，支持线性导航

### 1.2 页面内容规划

#### 1.2.1 Minecraft路径设置页
- 包含路径输入框和浏览按钮
- 显示路径说明文本
- 实时验证路径有效性

#### 1.2.2 Java设置页
- 从SettingsPage.xaml原封不动复制Java设置控件
- 包括Java版本列表、添加/刷新按钮
- Java选择方式（自动/手动）
- Java路径输入和浏览功能

#### 1.2.3 角色选择页
- 提供角色皮肤预览
- 角色名称输入
- 简单的角色创建界面

## 2. 技术实现方案

### 2.1 文件结构
- **Views/TutorialPage.xaml**：教程页面UI
- **ViewModels/TutorialPageViewModel.cs**：教程页面逻辑
- **Services/SettingsService.cs**：设置保存服务（复用现有）

### 2.2 导航逻辑
- 使用SelectorBar的SelectionChanged事件切换页面
- 底部按钮实现线性导航
- 记录用户当前位置，支持返回上一页

### 2.3 设置保存机制
- 直接使用现有的设置保存逻辑
- 路径设置和Java设置与主应用设置共享
- 确保设置更改即时生效

### 2.4 首次启动检测
- 在App.xaml.cs或ShellPage.xaml.cs中添加首次启动检测
- 使用本地设置存储教程完成状态
- 首次启动时自动导航到教程页面

## 3. UI设计规范

### 3.1 样式复用
- 使用应用已有的样式资源
- 遵循WINUI3设计指南
- 保持与现有设置页面一致的视觉风格

### 3.2 响应式设计
- 支持不同屏幕尺寸
- 控件布局自适应
- 确保在各种设备上都有良好的显示效果

## 4. 实现步骤

1. 创建TutorialPage.xaml和TutorialPageViewModel.cs
2. 实现SelectorBar导航结构
3. 添加三个页面的UI内容
4. 实现设置绑定和保存逻辑
5. 添加首次启动检测逻辑
6. 测试所有功能流程
7. 优化UI细节和用户体验

## 5. 预期效果

- 新用户首次启动时自动显示教程
- 清晰的步骤引导用户完成必要设置
- 设置更改即时保存并影响主应用
- 符合WINUI3设计规范的美观界面
- 支持线性导航和自由切换
- 完成后不再显示（可在设置中重新打开）
