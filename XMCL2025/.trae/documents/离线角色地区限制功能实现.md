# 离线角色地区限制功能实现

## 1. 需求分析

* 当用户启动游戏，选择的角色是离线角色时，检测当前用户的地区是否为中国大陆

* 非中国大陆用户不可使用离线登录

* 非中国大陆用户试图使用离线模式登录时，弹出弹窗，显示"当前地区无法使用离线登录，请添加微软账户登录。"

* 弹窗包含"前往"按钮，点击后跳转至角色页面

## 2. 技术实现方案

### 2.1 地区检测

使用系统区域设置检测地区，具体实现：

* 通过 `CultureInfo.CurrentCulture.Name` 或 `RegionInfo.CurrentRegion.TwoLetterISORegionName` 检测地区

* 中国大陆地区代码："CN" 或 "zh-CN"

### 2.2 离线角色判断

* 检查当前活跃角色的 `IsOffline` 属性

* `IsOffline` 为 `true` 表示离线角色

### 2.3 游戏启动流程修改

在 `启动ViewModel.cs` 的 `LaunchGameAsync` 方法中添加地区检查：

1. 检查当前选中角色是否为离线角色
2. 检测当前地区是否为中国大陆
3. 若为非中国大陆地区且使用离线角色，显示弹窗
4. 否则继续启动流程

### 2.4 弹窗实现

使用 `ContentDialog` 实现弹窗：

* 标题："地区限制"

* 内容："当前地区无法使用离线登录，请添加微软账户登录。"

* 主按钮："前往"（PrimaryButtonText）

* 次要按钮："取消"（CloseButtonText）

### 2.5 页面跳转

* 在弹窗的 `PrimaryButtonClick` 事件中，使用 `NavigationService` 跳转至角色页面

* 角色页面的路由："角色"

## 3. 代码修改点

### 3.1 启动ViewModel.cs

* 在 `LaunchGameAsync` 方法开始处添加地区和角色检查

* 实现弹窗显示逻辑

* 处理弹窗按钮点击事件

### 3.2 导航服务使用

* 注入 `INavigationService` 到 `启动ViewModel`

* 使用 `NavigationService.NavigateToAsync("角色")` 实现页面跳转

### 3.3 弹窗样式

* 遵循WinUI 3最佳实践

* 设置正确的 `XamlRoot`

* 确保弹窗在主线程上显示
