# 本地化实现计划

## 1. 概述
实现应用的本地化功能，支持简体中文和英文切换，将所有硬编码中文转换为资源文件引用。

## 2. 核心实现步骤

### 2.1 添加语言设置服务
- 创建 `LanguageSelectorService` 类，负责管理应用语言设置
- 实现语言加载、保存和切换功能
- 注册为单例服务

### 2.2 在设置页面添加语言选项
- 在设置页面的"个性化"标签页中添加语言选择区域
- 使用 RadioButton 或 ComboBox 实现中文/英文选择
- 绑定到 `SettingsViewModel` 中的语言属性

### 2.3 实现语言资源管理
- 扩展现有资源文件，添加所有页面需要的本地化字符串
- 确保每个硬编码中文都有对应的资源键
- 维护中英文资源文件的同步

### 2.4 替换硬编码中文
- 将所有 XAML 页面中的硬编码中文替换为 `x:Uid` 引用
- 将 ViewModel 和代码中的硬编码中文替换为资源引用
- 实现动态语言切换，确保设置更改后界面实时更新

### 2.5 测试和验证
- 验证所有页面在中英文切换下显示正常
- 确保所有功能在不同语言下正常工作
- 检查资源文件的完整性和一致性

## 3. 技术实现细节

### 3.1 语言选择服务
```csharp
public class LanguageSelectorService : ILanguageSelectorService
{
    // 实现语言管理功能
}
```

### 3.2 设置页面语言选项
- 在 SettingsPage.xaml 中添加语言选择控件
- 在 SettingsViewModel 中添加语言相关属性和命令

### 3.3 资源引用方式
- XAML: `x:Uid="ResourceKey"`
- 代码: `ResourceLoader.GetForCurrentView().GetString("ResourceKey")`

## 4. 实施顺序
1. 创建语言选择服务
2. 在设置页面添加语言选项
3. 替换设置页面的硬编码中文
4. 依次替换其他页面的硬编码中文
5. 测试语言切换功能
6. 完善资源文件

## 5. 预期效果
- 应用支持简体中文和英文切换
- 所有界面文本通过资源文件管理
- 语言设置实时生效
- 符合 WinUI 3 本地化最佳实践
