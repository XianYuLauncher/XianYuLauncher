# 微软账户令牌自动续签功能实现

## 1. 实现目标
- 在游戏启动时检测微软账户的accessToken是否过期
- 如果过期，自动进行续签
- 没联网时跳过检测
- 续签期间在InfoBar上显示状态信息
- 续签完毕后才开始启动游戏

## 2. 实现思路

### 2.1 令牌检测逻辑
- 在游戏启动流程中，在准备游戏文件之前添加令牌检测步骤
- 检查当前选中的角色是否为在线角色
- 如果是在线角色，计算令牌剩余有效期
- 如果剩余有效期小于1小时，触发令牌刷新

### 2.2 令牌刷新机制
- 调用现有的 `ForceRefreshTokenAsync()` 方法刷新令牌
- 在刷新过程中更新InfoBar显示
- 处理刷新成功和失败的情况

### 2.3 启动流程调整
1. 检查版本选择
2. 检查地区限制
3. 开始启动流程
4. **令牌检测与刷新**（新增）
5. 准备游戏文件
6. 构建Classpath
7. 启动游戏

## 3. 具体实现步骤

### 3.1 修改 `启动ViewModel.cs`

1. **添加令牌检查和刷新方法**
   ```csharp
   private async Task CheckAndRefreshTokenIfNeededAsync()
   {
       // 检查是否为在线角色
       if (SelectedProfile != null && !SelectedProfile.IsOffline)
       {
           // 计算令牌剩余有效期
           var issueTime = SelectedProfile.IssueInstant;
           var expiresIn = SelectedProfile.ExpiresIn;
           var expiryTime = issueTime.AddSeconds(expiresIn);
           var timeUntilExpiry = expiryTime - DateTime.UtcNow;
           
           // 如果剩余有效期小于1小时，刷新令牌
           if (timeUntilExpiry < TimeSpan.FromHours(1))
           {
               try
               {
                   // 显示InfoBar消息
                   IsLaunchSuccessInfoBarOpen = true;
                   LaunchSuccessMessage = $"{SelectedVersion} 正在进行微软账户续签...";
                   
                   // 调用令牌刷新方法
                   var 角色管理ViewModel = App.GetService<角色管理ViewModel>();
                   // 更新当前角色到角色管理ViewModel
                   角色管理ViewModel.CurrentProfile = SelectedProfile;
                   // 刷新令牌
                   await 角色管理ViewModel.ForceRefreshTokenAsync();
                   
                   // 刷新成功，更新当前角色信息
                   SelectedProfile = 角色管理ViewModel.CurrentProfile;
                   
                   // 更新InfoBar消息
                   LaunchSuccessMessage = $"{SelectedVersion} 微软账户续签成功...";
               }
               catch (Exception ex)
               {
                   // 刷新失败，提示用户
                   LaunchStatus = $"微软账户续签失败: {ex.Message}";
                   IsLaunchSuccessInfoBarOpen = false;
                   throw;
               }
           }
       }
   }
   ```

2. **在 `LaunchGameAsync()` 中调用令牌检查方法**
   ```csharp
   // 在准备游戏文件之前添加令牌检查
   await CheckAndRefreshTokenIfNeededAsync();
   
   // 5. 确保版本依赖和资源文件可用
   LaunchStatus = $"正在检查版本依赖和资源文件...";
   ```

3. **处理网络异常**
   - 在 `CheckAndRefreshTokenIfNeededAsync()` 方法中添加try-catch块
   - 捕获网络异常，跳过令牌刷新，继续启动流程

### 3.2 优化错误处理

- 在令牌刷新失败时，提供清晰的错误信息
- 允许用户选择重新登录或使用其他角色
- 在InfoBar上显示刷新结果

## 4. 实现注意事项

1. **线程安全**：确保在UI线程上更新InfoBar状态
2. **网络连接**：检查网络连接状态，没联网时跳过刷新
3. **用户体验**：提供清晰的状态反馈，避免用户困惑
4. **错误处理**：妥善处理刷新失败的情况，不影响游戏启动
5. **性能影响**：令牌检查和刷新过程应尽量快速，避免明显延迟

## 5. 测试计划

1. **正常流程测试**：使用有效的在线角色启动游戏，验证令牌检测和刷新功能
2. **过期令牌测试**：模拟过期令牌，验证自动刷新功能
3. **网络异常测试**：断开网络连接，验证跳过检测功能
4. **刷新失败测试**：模拟刷新失败，验证错误处理机制
5. **InfoBar显示测试**：验证刷新过程中InfoBar的状态显示

## 6. 预期效果

- 在线角色的令牌会在启动时自动检测和刷新
- 用户无需手动管理令牌有效期
- 刷新过程有清晰的状态反馈
- 网络异常时不影响游戏启动
- 刷新失败时提供明确的错误信息

通过实现这个功能，可以提高用户体验，减少因令牌过期导致的游戏启动失败，同时保持启动流程的顺畅。
