## 统一版本信息获取方案

### 问题分析

当前启动器在多个地方使用重复的逻辑来获取版本信息：

1. 尝试从XianYuL.cfg读取
2. 读取失败则从版本名提取
3. 缺乏对第三方启动器配置文件的支持
4. 自定义版本名称无法正确识别

### 解决方案

设计一个统一的版本信息获取服务，支持多来源配置文件读取和优雅的回退机制。

### 实现步骤

1. **VersionConfig类**

   * 确保与现有配置兼容

2. **创建统一的版本信息服务**

   * 新增`IVersionInfoService`接口，定义统一的版本信息获取方法

   * 实现`VersionInfoService`类，提供具体实现

   * 支持从多个配置文件来源读取信息

3. **实现多来源配置文件读取**

   * 支持从XianYuL.cfg读取

   * 支持从第三方启动器配置文件读取（如MultiMC、HMCL等）

   * 实现优先级机制，确保正确的读取顺序

4. **统一版本名提取逻辑**

   * 集中实现从版本名提取信息的逻辑

   * 支持多种版本名称格式

   * 提高识别准确性

5. **替换现有重复逻辑**

   * 更新所有使用版本信息的地方，调用统一服务

   * 确保向后兼容

   * 移除重复代码

6. **添加单元测试**

   * 测试不同配置文件来源的读取

   * 测试版本名提取逻辑

   * 测试错误处理和回退机制

### 核心方法设计

```csharp
// 获取版本配置信息
Task<VersionInfoResult> GetVersionInfoAsync(string versionId, string versionDirectory);

// 从版本目录读取配置文件
VersionConfig ReadVersionConfigFromDirectory(string versionDirectory);

// 从版本名提取信息
VersionConfig ExtractVersionInfoFromName(string versionId);
```

### 优先级顺序

1. 读取XianYuL.cfg（现有配置文件）
2. 读取第三方启动器配置文件（如MultiMC的instance.cfg、HMCL的version.json等）
3. 从版本JSON文件中提取信息
4. 从版本名称中提取信息

### 预期收益

* 统一的版本信息获取逻辑，减少代码重复

* 支持多种配置文件格式，提高兼容性

* 更好的错误处理和回退机制

* 便于扩展支持更多第三方启动器

* 提高版本信息识别的准确性
